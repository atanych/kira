## Log
- [[2026-07-13]] [[embeddings]] Embeddings landscape 2025-26. **Dense** (один вектор фикс. размера) — sentence-transformers, OpenAI text-embedding-3, Voyage, Cohere, bge, nomic. Для семантики. **Sparse** — SPLADE, BM25-как-вектор — ловят точные термины/редкие слова. Гибрид dense+sparse почти всегда бьёт чистый dense на -5-15% recall. **Multi-vector / late interaction** — ColBERT, ColPali (по вектору на токен, точнее но хранилище ×100). Для документов с картинками — ColPali топ (2024). Модальности: текст, код (CodeBERT, Voyage-code-3), мультимодал (CLIP, SigLIP, ImageBind), домен (BiomedBERT, LegalBERT). Архитектурные тренды: **Matryoshka** (bge-m3, OpenAI-3) — один вектор, truncate 3072→512 без потерь; **instruction-tuned** (e5-instruct, instructor, Voyage-3) — префикс 'represent this for search:' даёт boost, по дефолту юзать; **late chunking** (2024) — эмбеддишь весь док, потом режешь на чанки, контекст не теряется. Бенчмарк — MTEB leaderboard на HuggingFace.
- [[2026-07-13]] [[embeddings]] BM25 (Best Match 25) — старая формула ранжирования 1994, основа Elasticsearch/Lucene/Postgres FTS. Умный TF-IDF на стероидах. **Три компонента:** (1) TF с saturation — 10 вхождений ≠ 10× вес, кривая на плато (не спамишь ключами); (2) IDF — редкие слова весят больше; (3) length normalization — короткий док с совпадением важнее длинного. **BM25 vs embeddings:** BM25 = лексический (точное совпадение), embeddings = семантический (синонимы/смысл). Гибрид BM25 + dense через **RRF (Reciprocal Rank Fusion)** стабильно даёт +10-20% recall против чистого dense — стандарт RAG 2023-24. Postgres из коробки: `tsvector` + `ts_rank_cd` близко к BM25. Плюсы: без модели/GPU/обучения, точные термины (имена/коды/аббревиатуры) ловит идеально. Минусы: синонимы не понимает, опечатки убивают (нужен fuzzy).
